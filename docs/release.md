# prism release process

This repository releases from GitHub Actions.

## Automatic version tags

On a push to `main` or `master`, `.github/workflows/release.yml` compares the current `Cargo.toml` package version with the previous commit. If the version changed and tag `v<version>` does not already exist, the workflow creates that tag and continues with the release.

Manual `workflow_dispatch` also releases the current `Cargo.toml` version and creates `v<version>` when missing. Direct pushes of `v*` tags release that tag.

## Release gates

Before publishing artifacts, the workflow runs:

```sh
cargo fmt --check
cargo clippy --all --benches --tests --examples --all-features -- -D warnings
cargo test --all-features
cargo test --no-default-features
cargo build --release --locked --all-features
python3 scripts/release_smoke.py --bin target/release/prism
python3 scripts/perf_matrix.py --build --iterations 5 --warmups 1 --output target/perf-release-smoke.md
```

The performance smoke report is uploaded as a workflow artifact. It is a broad smoke matrix, not a deep benchmark run.

## Release artifacts

The release build matrix attaches:

| Artifact                          | Target                       | Notes                                           |
| --------------------------------- | ---------------------------- | ----------------------------------------------- |
| `prism-linux-amd64-musl.tar.gz`   | `x86_64-unknown-linux-musl`  | Static Linux binary, smoke-tested in CI.        |
| `prism-linux-aarch64-musl.tar.gz` | `aarch64-unknown-linux-musl` | Cross-built static Linux binary.                |
| `prism-darwin-aarch64.tar.gz`     | `aarch64-apple-darwin`       | macOS Apple Silicon binary, smoke-tested in CI. |
| `prism-windows-amd64.zip`         | `x86_64-pc-windows-msvc`     | Windows binary, smoke-tested in CI.             |
| `SHA256SUMS`                      | all release files            | SHA-256 checksums generated on Ubuntu.          |

Release notes are generated by GitHub from the tag history.

## Local release smoke

After building a release binary locally:

```sh
cargo build --release --locked --all-features
python3 scripts/release_smoke.py --bin target/release/prism
```

Inside the Nix shell, the wrapper is available as:

```sh
release_smoke --bin target/release/prism
```
